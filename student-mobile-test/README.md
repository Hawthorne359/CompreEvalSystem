# 学生端手机测试（同一 WiFi 下用手机访问本机服务）

本文件夹**独立于主项目**，只提供「让手机在局域网内访问你电脑上跑的前端/后端」的说明和脚本，**不会修改** `frontend/` 或 Django 主代码。

---

## 一、思路说明（能不能做？能）

- 你现在在电脑上访问的是 **127.0.0.1:5173**（前端）和 **127.0.0.1:8000**（后端）。  
  **127.0.0.1** 只表示「本机」，手机无法通过这个地址找到你的电脑。

- 同一 WiFi 下，电脑有一个**局域网 IP**（一般是 `192.168.x.x`）。  
  只要让**前端开发服务器**监听在所有网卡上（`0.0.0.0`），手机用浏览器的地址栏访问 **http://你的电脑局域网IP:5173**，就能打开和你电脑上一样的页面。

- **请求怎么到后端？**  
  前端 Vite 已配置了代理：`/api`、`/media` 会转发到 `127.0.0.1:8000`。  
  手机只和「你电脑上的 Vite」通信；Vite 再在你这台电脑上请求 Django。所以：
  - **只需让 Vite 监听 0.0.0.0**（手机能连上 5173）；
  - **Django 可以继续只监听 127.0.0.1:8000**，不必改 Django 的 `runserver` 或 `ALLOWED_HOSTS`。

- **结论**：  
  - 不崩项目：主项目里可以不改任何代码，只用「另一种启动方式」跑前端。  
  - 单独文件夹：把「手机测试」的说明和脚本都放在本目录，主项目保持原样。

## 二、你需要做的（三步）

### 1. 启动后端（和平时一样）

在项目**根目录**（有 `manage.py` 的目录）执行：

```bash
.\.venv\Scripts\python.exe manage.py runserver 127.0.0.1:8000
```

保持这个窗口不关。

### 2. 用「允许局域网访问」的方式启动前端

再开一个终端，进入前端目录，加 `--host` 让 Vite 监听 `0.0.0.0`：

```bash
cd c:\Users\Hawthorne\Desktop\biyesheji\CompreEvalSystem\frontend
npm run dev -- --host
```

终端里会多出一行类似：

```text
  Network:  http://192.168.x.x:5173/
```

这个 **192.168.x.x** 就是你这台电脑在当前 WiFi 下的局域网 IP。

### 3. 手机连同一 WiFi，浏览器访问

手机连上**和电脑同一个 WiFi**，在浏览器地址栏输入：

```text
http://192.168.x.x:5173
```

（把 `192.168.x.x` 换成你终端里显示的 Network 地址）

即可用手机打开学生端页面，登录后测试「我的提交」、拍照上传等。

## 三、如何查电脑的局域网 IP（若终端没显示）

- **Windows（PowerShell 或 CMD）**：  
  ```bash
  ipconfig
  ```  
  看 **无线局域网适配器 WLAN** 下的 **IPv4 地址**，即 192.168.x.x。

- 本目录下的 **`show-lan-ip.ps1`** 会在控制台输出本机 LAN IP 和访问地址，方便复制到手机。  
- **`start-frontend-mobile.ps1`** 会自动进入 `frontend` 并执行 `npm run dev -- --host`，无需自己敲命令（后端仍需在项目根目录另开终端先启动）。

## 四、常见问题

| 问题 | 处理 |
|------|------|
| 手机打不开页面 | 确认手机和电脑在同一 WiFi；电脑防火墙若弹出「允许 Node 访问网络」要点允许。 |
| 页面能开但接口 404 / 报错 | 确认后端已用 `runserver 127.0.0.1:8000` 启动，且前端是用 `npm run dev -- --host` 启动的（这样 /api 会由 Vite 转发到本机 8000）。 |
| 换了一个 WiFi | 重新执行 `npm run dev -- --host`，看终端里新的 Network 地址，或再运行一次 `show-lan-ip.ps1`。 |

## 五、和主项目的关系

- **主项目**：照常用 `npm run dev`（不加 `--host`）在电脑上开发即可。  
- **手机测试**：仅在你需要时，用 `npm run dev -- --host`，并到本目录看说明或跑 `show-lan-ip.ps1` 查 IP。  
- 本目录只有说明和辅助脚本，**不包含**前端/后端源码，也不会改主项目配置，不会把项目搞崩。

# 双评主流程校正回归矩阵

## 目标

- 验证“双评助理主评 + 辅导员按模式介入”流程与配置一致。
- 验证“关闭双评后进入单评模式”时的任务分配、评分与仲裁语义一致。
- 验证 `same_major + 打散` 可正常生成任务并可追踪。
- 验证权限收口：未分配不可评分、未超阈值不可仲裁。
- 验证“模板复用、模块阈值、辅导员放行分发”三项新增能力不回归。

## 组合矩阵

| ---------------------- | --- | ---------------- | ---- | ------------------ |
| same_class             | 关   | arbitration_only | 2    | 每份仅2条助理任务，无辅导员常规任务 |

## 单评矩阵（dual_review_enabled=false）

| 单评执行人             | 任务生成预期                | 评分约束                         | 仲裁预期                  |
| ----------------- | --------------------- | ---------------------------- | --------------------- |
| assistant_single  | 每份提交仅1条助理任务（round=1）  | 同一提交同一指标仅允许1条常规评分记录          | 允许异常上报辅导员仲裁           |
| counselor_single  | 每份提交仅1条辅导员任务（round=1） | 同一提交同一指标仅允许1条常规评分记录          | 可按管理权限发起仲裁（通常用于人工复核） |

## 混合分发矩阵（mixed-by-mode）

| 模式 | 生成任务后首站 | 辅导员动作 | 预期结果 |
| ---- | ---- | ---- | ---- |
| dual_review_enabled=true | 辅导员分发池（`counselor_dispatch`） | 放行 | 助理任务落地，辅导员分发任务转 `released` |
| dual_review_enabled=false + assistant_single | 辅导员分发池（`counselor_dispatch`） | 放行 | 单评助理任务落地（每提交1人） |
| dual_review_enabled=false + counselor_single | 辅导员常规任务 | 无放行 | 仅辅导员可评分，下放不可用 |

## 阈值矩阵（模块 + 总分）

| 场景 | 配置 | 预期 |
| ---- | ---- | ---- |
| 仅通用阈值 | `score_diff_threshold=5` | 沿用旧逻辑，按指标分差判定 |
| 模块阈值覆盖 | `module_diff_thresholds={A1:3}` + `score_diff_threshold=5` | A1按3，其余模块按5 |
| 总分阈值生效 | `overall_score_diff_threshold=8` | 任一指标发起仲裁前，允许按总分差命中 |
| 模块/总分同时配置 | 两者均配置 | 任一命中即可放行仲裁 |

## 模板复用矩阵

| 模板保存项 | 应用目标 | 预期 |
| ---- | ---- | ---- |
| basic + review | 新项目 | 仅基础信息与评审规则覆盖 |
| indicator + weight | 草稿项目（无提交） | 指标与权重覆盖成功 |
| indicator | 已有提交项目 | 被阻止并提示“禁止覆盖指标结构模板” |


## 权限校验用例

- 助理使用标准评分接口提交：应拒绝（需走助理评分接口）。
- 非任务归属用户提交评分：应拒绝。
- 评审老师对非管辖班级提交仲裁：应拒绝。
- 分差未超阈值时发起仲裁：应拒绝。
- 单评模式下出现第2条常规评分写入：应拒绝。
- 辅导员未放行前助理无可评分任务：应符合预期。
- 非模板创建人套用私有模板：应拒绝。

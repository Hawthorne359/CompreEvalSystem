# 补交通道批量推送说明（现行实现）

## 背景
管理员在 `系统管理 > 补交通道管理 > 通道列表` 中，需要看到每个通道已收集多少待推送补交记录，并在合适时机批量推送给评阅老师处理，避免零散打扰评审。

## 当前接口与页面逻辑

### 管理端接口
- `GET /api/v1/admin/late-channels/`
  - 查询通道列表。
  - 支持 `active_only=1` 仅返回激活通道。
  - 支持 `include_submissions=1` 时返回每个通道的 `pending_submissions` 明细。
- `POST /api/v1/admin/late-channels/<id>/batch-push/`
  - 批量推送补交记录到评审流程。
  - 可选参数 `submission_ids`，用于“只推送选中的记录”。

### 前端页面
- 页面文件：`frontend/src/views/SystemAdmin.vue`
- API 文件：`frontend/src/api/admin.js`
- 在 `通道列表` 中：
  - 每条通道显示 `pending_submissions_count`（待推送数量）。
  - 点击“查看”可展开待推送记录明细。
  - 支持全量推送或勾选后推送部分记录。

## 数据口径
- 待推送记录定义：
  - `via_late_channel=True`
  - `status='submitted'`
  - 且匹配通道范围（用户/班级）与项目限定（若通道限定了项目）。
- 批量推送动作：
  - 将上述记录状态批量更新为 `under_review`。
  - 推送后评阅老师在评审任务中处理。

## 近期修复点（2026-03）
- 修复通道列表展开行作用域问题，避免列表异常渲染。
- 修复 `include_submissions=1` 序列化上下文遗漏，恢复补交记录可见性。
- 优化待推送数量计算为批量聚合，降低通道列表加载耗时。
- 增加通道列表空态与失败提示，减少“看起来一直加载中”的误判。

## 说明
- 历史方案中提到的 `GET /admin/projects/<id>/batch-push-late/` 与 `getLateSubmissionsPending` 已不再使用。
- 当前统一以“通道”为中心进行查看与推送。
